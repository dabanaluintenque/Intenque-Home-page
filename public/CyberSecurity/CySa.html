Identity and Acces Management

Multi-Factor Authentication (MFA)
A way to prove you say you are, by something you have, something you are, something you know.

Single Sign-On (SSO)
Authentication process that allow a single user access to multiple systems, apps, websites. Single set of Credentials. By authenticating one time.

Federation
- OpenID
- Transitive trust (if A trust B and C then trust A)
- Shibboleth:
Is a federated identity method, which is based on SAML (Secure access, markup language). It's Often used by universities, Colleges, or public service organizations. Where user attent to retribute resources from a website and that website sends SAML authentication information over URL queries. Then the user is redirected to an identiy provider with which they can authenticate using the SAML information.
It's a different way of accomplishing the samething as OpenID process.

Privileged Access Management (PAM)
Refers to the use of people, application, services, processes, and technology to control, secure, monitor, and audit identities. Allow you track who access what. Greater awareness and visibility into credentials used in our environment.

Passwordless
It's a type of the authentication method where users can access an account without entering a password. It relies on one of the other identity mechanisms in multi-factor authentication.
Here you can only use semething that you are, like biometric data, or something that you have, one time Code.

Cloud Access Security Broker (CASB)
Cloud SOC CASB
Enterprise management software that is designed to mediate access to the cloud.
CASB can enable SSO, it can enforce access control and what to do when authenticate, scan the device.
There's three way that CASB is implemented into an environment.
1. Forward Proxy: client Network Edge
2. Reverse Proxy: Cloud Network Edge
3. API (Application Programming Interface)
