Big O Notation

What is the idea of Big O notation?

Imagine we have multipe implementations of the same function.
How can we determine which one is the best?

Why Cares about Big O notation?

  • It's important to have a precise vocabulary to talk about how our code performs
  • Useful for discussing trade-offs between different approaches
  • When your code slows down or crashes, identifying parts of the code that are inefficient can help us find pain points in our applications
  • It comes up in technical interivews!

Examples

Suppose we want to write a function that calculates the sum of all numbers from 1 up to n (including n);

Example #1

function addUpTo (n) {
let total = 0;
for (let i = 1; i <= n; i ++) {
total += i;
}
return total;
}

Example # 2

function addUpTo2(n) {
return n * (n + 1) / 2;
}

Before we dicide which one is better first we have to understand what does better mean?

  • Faster?
  • Less memory-intensive?
  • More readable?

All of the above is important but we will focus on faster. (The time that it's take to run a code)

We can use the JavaScript built in function "performance.now()" to check the time but is not the most releable way to determine the faster code.

Here is the example of using performance.now with our two examples above

var startingTime = performance.now();
addUpTo2(1000000000);
var endingTime = performance.now();
var mellisecons = 1000;
console.log(`starting time ${startingTime}`);
console.log(`ending time ${endingTime}`);
console.log(`Total running time: ${(endingTime - startingTime) / mellisecons} seconds.`);

The Problem with Time

  • Different machines will record different times
  • The same machine will record different times!
  • For fast algorithms, speed measurements may not be precise enough?

So if we not using time, What should we use then?
